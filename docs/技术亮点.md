## 亮点 1：两次采样差值法计算 CPU 使用率

**问题**：`/proc/stat` 记录的是系统启动以来的**累计** CPU jiffies，不是瞬时使用率。

**方案**：取两个时间点的快照，计算 delta 差值。

```bash
# 第一次采样
sample1=$(awk 'NR==1 {print $2+$3+$4+$5+$6+$7+$8+$9, $5+$6}' /proc/stat)
sleep 0.5
# 第二次采样
sample2=$(awk 'NR==1 {print $2+$3+$4+$5+$6+$7+$8+$9, $5+$6}' /proc/stat)
# CPU% = (delta_total - delta_idle) / delta_total × 100
```

**延伸**：各核心使用率同理，读取 `/proc/stat` 的 `cpu0`, `cpu1`, ... 行，同样的算法。

## 亮点 2：管道实现实时日志流式处理

**问题**：如何在不把整个文件读入内存的情况下，实时过滤和标记日志？

**方案**：Shell 管道实现流式 IPC，数据不需要落盘。

```bash
tail -f /var/log/syslog | grep --color=auto -E "ERROR|FAIL|CRITICAL"
```

**OS 原理**：内核管道缓冲区（默认 64KB）连接两个进程。`tail -f` 监控 inode 变化（inotify 机制），一旦有新数据写入，立即通过管道传给 `grep`。生产者-消费者模型，无需轮询。

## 亮点 3：SUID/SGID 权限位安全审计

**问题**：Linux 系统中哪些文件存在提权风险？

**方案**：扫描关键目录，检测特殊权限位。

```bash
# World-Writable: 任何人可写
find /etc /bin /sbin -type f -perm -o+w

# SUID: 执行时以文件所有者身份运行 (chmod 4xxx)
find /etc /bin /sbin -type f -perm -4000

# SGID: 执行时以文件所属组身份运行 (chmod 2xxx)
find /etc /bin /sbin -type f -perm -2000
```

**OS 原理**：Linux 文件权限模型中的特殊位。SUID 文件如 `/usr/bin/passwd` 是合法的，但如果 `/tmp/backdoor` 有 SUID root 位，就是安全漏洞。

## 亮点 4：信号驱动的优雅退出机制

**问题**：实时日志追踪模式下按 Ctrl+C，守护进程被 kill，如何保证资源正确释放？

**方案**：`trap` 捕获信号 + 清理函数。

```bash
cleanup() {
    rm -f /tmp/sysops_*.tmp
    log_info "收到退出信号，清理完成"
    exit 0
}
trap cleanup SIGINT SIGTERM
```

**OS 原理**：Ctrl+C → 内核向进程发送 SIGINT (信号 2)，`kill` 命令发送 SIGTERM (信号 15)。`trap` 注册信号处理器，在执行清理逻辑后才退出，避免临时文件残留和资源泄露。

## 亮点 5：HTML 报告 + 系统健康评分算法

**问题**：如何将四个模块的文本输出整合为一个直观的可视化报告？

**方案**：纯 Shell 拼接 HTML 模板 + 加权评分算法。

```
健康评分 = 100 - CPU超出扣分 - 内存超出扣分 - 磁盘超出扣分
         = 100 - (cpu-80)×2 - (mem-80)×1.5 - (disk-90)×2
最低 0 分，最高 100 分
≥80: 优秀 | ≥60: 良好 | ≥40: 一般 | <40: 警告
```

**OS 原理**：跨模块数据聚合 + 自动化报告生成体现了运维自动化中"监控→分析→报告→响应"的完整工作流。
